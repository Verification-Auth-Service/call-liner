# AI Program Rules

このリポジトリで AI がコードを追加・修正するときは、以下を必須とする。

## 必須ルール

1. コメントを書くこと

- 条件分岐（`if` / `switch`）には、分岐理由が分かるコメントを付ける。
- 公開関数には、入力例・出力例を含む説明コメントを付ける。

2. テストを書くこと

- 仕様追加・挙動変更を伴う修正には、必ず自動テストを追加または更新する。
- 正常系だけでなく、最低 1 つの異常系（エラー/バリデーション）を含める。

3. 検証してから完了報告すること

- `pnpm typecheck` と `pnpm test` を実行し、結果を報告する。
- 失敗している場合は原因と未解決事項を明記する。
